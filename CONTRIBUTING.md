# Contributing

## Before opening a pull request

1. Keep changes focused and preserve the two existing widget styles.
2. Do not add local Codex session data, built apps, ZIP files, or screenshots containing personal information.
3. Build the affected style before submitting:

```zsh
zsh build.sh doodle
zsh build.sh glass
```

4. Describe visible behavior changes and the macOS version used for verification.

## Issues

For quota-reading issues, include the widget's visible state and error details without attaching raw files from `~/.codex/sessions`.
